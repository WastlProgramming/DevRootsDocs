Warum Typescript

TypeScript ist so Ähnlich wie Javascript. Es ist eine Erweiterung von Javascript. TypeScript ist eine Programmiersprache, die auf JavaScript aufbaut. Sie ist typisiert und objektorientiert. TypeScript ist eine von Microsoft entwickelte Programmiersprache, die auf JavaScript aufbaut.

Später wird TypeScript in JavaScript übersetzt. WICHTIG TypeScript unterstützt Javascript heißt du kannst ganz normal Javascript in Typescript verwenden.

Typen

Folgende Typen gibt es in TypeScript:

  • number: Ganzzahlen und Fließkommazahlen

  • string: für Zeichenketten

  • boolean: für Wahrheiten

  • any: für beliebige Werte

  • null: für Null Werte

  • undefinied: für undefinied Werte // ist sicherer als wie any

Der Typ any ist ein dynamischer Typ, der alle Typen zulässt. Er ist mit dem Typ Object in Java vergleichbar. wir können diesen Variablen jeden Wert zuweisen.

let x : string
let y : number

Strukturierte Typen sind komplexere Datentypen, die mehrere Werte speichern können. Dazu gehören:

  • object für Objekte

  • tuple für Tulpen

  • interface für schnittstellen

  • class für Klassen

Arrays

Typescripthat eine Spezelle Syntax für typing Arrays.

const names: string[] = []
names.push("Dylan")
// names.push(3); // Error: Argument of type 'number' is not assignable to parameter of type 'string'.

Typinferenz

Typescript kann den tTyp eines Arrays ableiten, wenn es Werte enthält

const numbers = [1, 2, 3]; // inferred to type number[]
numbers.push(4); // no error
// comment line below out to see the successful assignment
numbers.push("2"); // Error: Argument of type 'string' is not assignable to parameter of type 'number'.
let head: number = numbers[0]; // no error

Tuples

Tuples

Ein Tupel ist ein typisiertes Array mit einem Pre -definierte Länge und Typen für jeden Index.

Tupel sind großartig, weil sie ermöglichen, dass jedes Element im Array ein bekannter Werttyp ist.

Um ein Tupel zu definieren, geben Sie den Typ jedes Elements im Array an:

// define our tuple
let ourTuple: [number, boolean, string];

// initialize correctly
ourTuple = [5, false, 'Coding God was here'];

wenn wir die Reihenfolge vertauschen folgt dies zu einen Error

Readonly Tupel

// define our readonly tuple
const ourReadonlyTuple: readonly [number, boolean, string] = [5, true, 'The Real Coding God'];
// throws error as it is readonly.
ourReadonlyTuple.push('Coding God took a day off');

Hier kommt ein Fehler lassen wir readonly weg schreibt er hier einfach danch noch einen String rein.

Benannte Tupel

Benannte Tupel ermöglichen es uns, an jedem Index Kontext für unsere Werte bereitzustellen.

const graph: [x: number, y: number] = [55.2, 41.3];

Tupel zerstören

Da Tupel Arrays sind, können wir sie auch destrukturieren.

const graph: [number, number] = [55.2, 41.3];
const [x, y] = graph;

Objekte Types

TypeScript bietet eine spezielle Syntax für die Typisierung von Objekten.

const car = {
    type: string,
    model: string,
    color: string,
    year: number,
    price: number
  } = {
    type: "Fiat",
    model: "500",
    color: "white",
    year: 2010,
    price: 10000
  };

  // Die Anweisung `console.log("hallo")` in die Objektdefinition von `car` verschieben
  console.log(car.type);
  console.log(car.model);
  console.log(car.color);
  console.log(car.year);
  console.log(car.price);

Nicht wunrdern das Fehler angezeigt werden es sind keine Fehler

Wenn kein Datentyp zugewiesen wird, wird der Typ any verwendet und kann beliebig geändern werden. Wenn aber ein anderer Datentyp zugewiesen wird, kann dieser nicht mehr geändert werden.

Ohne Datentyp

  const car2 = {
    type: "fiat"
  }

  car2.type = 2
  console.log(car2.type)

Mit Datentyp

  const car2 = {
    type: string
  }

  car2.type = "Fiat"
  car2.type = 2
  console.log(car2.type)

Der Fehler tritt erst beim umwandeln auf.

Indexsignaturen

Indexsignaturen können für Objekte ohne definierte Liste von Eigenschaften verwendet werden.

var nameAgeMap = {};
nameAgeMap.Jack = 25; // no error
nameAgeMap.Mark = "Fifty"; // Error: Type 'string' is not assignable to type 'number'.

In dem Code wird ein Objekt nameAgeMap mit einem Index-Signatur definiert. Ein Index-Signatur ist eine Möglichkeit, Eigenschaften eines Objekts mit einem Index statt mit einem Eigenschaftsnamen zu definieren. In diesem Fall definiert die Index-Signatur [index: string]: number, dass jede Eigenschaft im Objekt einen String-Schlüssel und einen Zahlenwert hat.

Der Code weist dann den Wert 25 der Eigenschaft mit dem Schlüssel "Jack" zu. Dies ist gültig, da der Schlüssel ein String und der Wert eine Zahl ist, was dem Typ der Index-Signatur entspricht.

Wenn du jedoch den Wert "Fifty" der Eigenschaft mit dem Schlüssel "Mark" zuweist, wirft TypeScript einen Fehler. Dies liegt daran, dass der Schlüssel ein String und der Wert ebenfalls ein String ist, was nicht mit dem Typ der Index-Signatur kompatibel ist. Die Index-Signatur legt fest, dass der Wert eine Zahl sein muss, und TypeScript erzwingt diese Regel.

UM dies zu Beheben:

const nameAgeMap: { [index: string]: string | number } = {};

nameAgeMap.Jack = 25; // Kein Fehler
nameAgeMap.Mark = "Fifty"; // Kein Fehler

Enums

Ein Enum // Aufzählung ist eine spezielle "Klasse" das eine Gruppe von Konstanten(unveränderliche Werte) enthält. Enums gibt es in zwei Varianten string und numeric.

Beginnen wir mit der numerischen Eingabe.

enum CardinalDirections {
  North,
  East,
  South,
  West
}
let currentDirection = CardinalDirections.North;
// logs 0
console.log(currentDirection);
// throws error as 'North' is not a valid enum
currentDirection = 'North'; // Error: "North" is not assignable to type 'CardinalDirections'.

Um dies zu Initialisieren müssen wir die Werte zuweisen.

enum CardinalDirections {
  North = 1,
  East,
  South,
  West
}
// logs 1
console.log(CardinalDirections.North);
// logs 4
console.log(CardinalDirections.West);

Wir können jeden einzelenen Wert eine Zahl zuweisen.

enum StatusCodes {
  NotFound = 404,
  Success = 200,
  Accepted = 202,
  BadRequest = 400
}
// logs 404
console.log(StatusCodes.NotFound);
// logs 200
console.log(StatusCodes.Success);

String Werte

enum CardinalDirections {
  North = 'North',
  East = "East",
  South = "South",
  West = "West"
};
// logs "North"
console.log(CardinalDirections.North);
// logs "West"
console.log(CardinalDirections.West);

Typaliasse und Schnittstellen

TypeScript ermöglicht die Definition von Typen getrennt von den Variablen, die sie verwenden.

Aliase und Schnittstellen ermöglichen die einfache gemeinsame Nutzung von Typen zwischen verschiedenen Variablen/Objekten.

Typaliase ermöglichen die Definition von Typen mit einem benutzerdefinierten Namen (einem Alias).

Typaliase können für Grundelemente wie string oder komplexere Typen wie objects und arrays verwendet werden:

type CarYear = number
type CarType = string
type CarModel = string
type Car = {
  year: CarYear,
  type: CarType,
  model: CarModel
}

const carYear: CarYear = 2001
const carType: CarType = "Toyota"
const carModel: CarModel = "Corolla"
const car: Car = {
  year: carYear,
  type: carType,
  model: carModel
};

Schnittstellen

Schnittstellen ähneln Typaliasen, außer dass sie nur für object Typen gelten.

interface Rectangle {
  height: number,
  width: number
}

const rectangle: Rectangle = {
  height: 20,
  width: 10
};

Eine Schnittstelle erweitern bedeutet, dass Sie eine neue Schnittstelle mit denselben Eigenschaften wie das Original und zusätzlich etwas Neuem erstellen.

interface Rectangle {
  height: number,
  width: number
}

interface ColoredRectangle extends Rectangle {
  color: string
}

const coloredRectangle: ColoredRectangle = {
  height: 20,
  width: 10,
  color: "red"
};

[[Union Types]] == Union Types

Union-Typen werden verwendet, wenn ein Wert mehr als einen einzelnen Typ haben kann.

Zum Beispiel, wenn eine Eigenschaft string oder number wäre.

Gewerkschaft | (ODER) Mit | sagen wir, dass unser Parameter ein string oder number ist:

function printStatusCode(code: string | number) {
  console.log(`My status code is ${code}.`)
}
printStatusCode(404);
printStatusCode('404');

Typischer Fehler

function printStatusCode(code: string | number) {
  console.log(`My status code is ${code.toUpperCase()}.`) // error: Property 'toUpperCase' does not exist ontype 'string | number'.
  Property 'toUpperCase' does not exist on type 'number'
}

Funktionen

TypeScript verfügt über eine spezielle Syntax zum Eingeben von Funktionsparametern und Rückgabewerten.

Rückgabetyp

// the `: number` here specifies that this function returns a number
function getTime(): number {
  return new Date().getTime();
}

Wenn kein Rückgabetyp definiert ist, versucht TypeScript, ihn anhand der Typen der zurückgegebenen Variablen oder Ausdrücke abzuleiten.

Unötiger Rückgabetyp

Der Typ void kann verwendet werden, um anzugeben, dass eine Funktion keinen Wert zurückgibt.

function printHello(): void {
  console.log('Hello!');
}

Parameter

function multiply(a: number, b: number) {
  return a * b;
}

Optionale Parameter

Optionale Parameter können mit einem ? nach dem Parameter definiert werden.

// the `?` operator here marks parameter `c` as optional
function add(a: number, b: number, c?: number) {
  return a + b + (c || 0);
}

Hier würde es in Javascript so aussehen

function add(a, b, c) {
    return a + b + (c || 0);
}

Standart Parameter

function pow(value: number, exponent: number = 10) {
  return value ** exponent;
}

Benannte Parameter

Die Eingabe benannter Parameter erfolgt nach demselben Muster wie die Eingabe normaler Parameter.

    function divide({ dividend, divisor }: { dividend: number, divisor: number }) {
    return dividend / divisor;
    }

Ruheparameter

Restparameter können wie normale Parameter typisiert werden, der Typ muss jedoch ein Array sein, da Restparameter immer Arrays sind.

function add(a: number, b: number, ...rest: number[]) {
  return a + b + rest.reduce((p, c) => p + c, 0);
}

Besonderes

Funktionstypen können getrennt von Funktionen mit Typaliasen angegeben werden.

type Negate = (value: number) => number;

// in this function, the parameter `value` automatically gets assigned the type `number` from the type `Negate`
const negateFunction: Negate = (value) => value * -1;

Casting

Bei der Arbeit mit Typen kann es vorkommen, dass der Typ einer Variablen überschrieben werden muss, beispielsweise wenn von einer Bibliothek falsche Typen bereitgestellt werden.

Casting ist der Prozess des Überschreibens eines Typs.

let x: unknown = 'hello';
console.log((x as string).length);

Durch das Casting wird der Typ der Daten innerhalb der Variablen nicht wirklich geändert. Beispielsweise funktioniert der folgende Code nicht wie erwartet, da die Variable x immer noch eine Zahl enthält.

let x: unknown = 4;
console.log((x as string).length); // prints undefined since numbers don't have a length

TypeScript wird weiterhin versuchen, Umwandlungen zu überprüfen, um Umwandlungen zu verhindern, die nicht korrekt erscheinen. Im Folgenden wird beispielsweise ein Typfehler ausgegeben, da TypeScript weiß, dass die Umwandlung einer Zeichenfolge in eine Zahl keinen Sinn ergibt ohne Konvertierung der Daten:

console.log((4 as string).length); // Error: Conversion of type 'number' to type 'string' may be a mistake because neither type sufficiently overlaps with the other. If this was intentional, convert the expression to 'unknown' first.

Casten mit <>

Mit <> funktioniert genauso wie das Casting mit as.

let x: unknown = 'hello';
console.log((<string>x).length);

Force Casting

Um Typfehler zu überschreiben, die TypeScript bei der Umwandlung auslösen kann, führen Sie zunächst eine Umwandlung in unknown und dann in den Zieltyp durch.

let x = 'hello';
console.log(((x as unknown) as number).length); // x is not actually a number so this will return undefined

Classes

TypeScript fügt JavaScript-Klassen Typen und Sichtbarkeitsmodifikatoren hinzu.

Mitglieder: Typen Die Mitglieder einer Klasse (Eigenschaften und Methoden) werden mithilfe von Typanmerkungen typisiert, ähnlich wie bei Variablen.

class Person {
  name: string;
}

const person = new Person();
person.name = "Jane";

Sichbarkeit

Klassenvariablen erhalten ausßerdem spezielle Modifikatoren, die die Sichtbarkeit der Variablen einschränken.

  • public: Die Variable ist von überall sichtbar.

  • private: Die Variable ist nur innerhalb der Klasse sichtbar.

  • protected: Die Variable ist nur innerhalb der Klasse und ihrer Unterklassen sichtbar.

class Person {
  private name: string;

  public constructor(name: string) {
    this.name = name;
  }

  public getName(): string {
    return this.name;
  }
}

const person = new Person("Jane");
console.log(person.getName()); // person.name isn't accessible from outside the class since it's private

Parametereigenschaften

TypeScript bietet eine praktische Möglichkeit, Klasseneigenschaften zu definieren und zu initialisieren, indem Sie sie direkt in den Konstruktorparametern definieren.

class Person {
  // name is a private member variable
  public constructor(private name: string) {}

  public getName(): string {
    return this.name;
  }
}

const person = new Person("Jane");
console.log(person.getName());

Schreibgeschützt

So ähnlich wie bei Arrays kann hier das Schlüsselwort readonly verwendet werden.

class Person {
  private readonly name: string;

  public constructor(name: string) {
    // name cannot be changed after this initial definition, which has to be either at it's declaration or in the constructor.
    this.name = name;
  }

  public getName(): string {
    return this.name;
  }
}

const person = new Person("Jane");
console.log(person.getName());

Vererbung: Implementiert

Schnittstellen (hier behandelt) können verwendet werden, um den Typ zu definieren, dem eine Klasse durch folgen muss Schlüsselwort.implements

interface Shape {
  getArea: () => number;
}

class Rectangle implements Shape {
  public constructor(protected readonly width: number, protected readonly height: number) {}

  public getArea(): number {
    return this.width * this.height;
  }
}

Eine Klasse kann mehrere Schnittstellen implementieren, indem sie jede nach implements auflistet, getrennt durch ein Komma wie folgt: class Rectangle implements Shape, Colored { …​

Vererbung : Erweitert

Klassen können sich gegenseitig durch das Schlüsselwort extends erweitern. Eine Klasse kann nur eine andere Klasse erweitern.

interface Shape {
  getArea: () => number;
}

class Rectangle implements Shape {
  public constructor(protected readonly width: number, protected readonly height: number) {}

  public getArea(): number {
    return this.width * this.height;
  }
}

class Square extends Rectangle {
  public constructor(width: number) {
    super(width, width);
  }

  // getArea gets inherited from Rectangle
}

Überschreiben

Überschreiben Wenn eine Klasse eine andere Klasse erweitert, kann sie die gleichnamigen Mitglieder der übergeordneten Klasse ersetzen.

Neuere Versionen von TypeScript ermöglichen die explizite Markierung mit dem Schlüsselwort override.

interface Shape {
  getArea: () => number;
}

class Rectangle implements Shape {
  // using protected for these members allows access from classes that extend from this class, such as Square
  public constructor(protected readonly width: number, protected readonly height: number) {}

  public getArea(): number {
    return this.width * this.height;
  }

  public toString(): string {
    return `Rectangle[width=${this.width}, height=${this.height}]`;
  }
}

class Square extends Rectangle {
  public constructor(width: number) {
    super(width, width);
  }

  // this toString replaces the toString from Rectangle
  public override toString(): string {
    return `Square[width=${this.width}]`;
  }
}

Standardmäßig ist das Schlüsselwort override beim Überschreiben einer Methode optional und hilft nur dabei, das versehentliche Überschreiben einer nicht vorhandenen Methode zu verhindern. Verwenden Sie die Einstellung noImplicitOverride, um die Verwendung beim Überschreiben zu erzwingen

Abstrakte Klassen

Klassen können so geschrieben werden, dass sie als Basisklasse für andere Klassen verwendet werden können, ohne dass alle Mitglieder implementiert werden müssen. Dies geschieht mit dem Schlüsselwort abstract. Mitglieder, die nicht implementiert werden, verwenden auch das Schlüsselwort abstract.

abstract class Polygon {
  public abstract getArea(): number;

  public toString(): string {
    return `Polygon[area=${this.getArea()}]`;
  }
}

class Rectangle extends Polygon {
  public constructor(protected readonly width: number, protected readonly height: number) {
    super();
  }

  public getArea(): number {
    return this.width * this.height;
  }
}

Generics

Mit Generika können „Typvariablen“ erstellt werden. die zum Erstellen von Klassen, Funktionen & Typaliase, die die von ihnen verwendeten Typen nicht explizit definieren müssen.

Generics erleichtert das Schreiben von wiederverwendbarem Code.

Funktionen

Generika mit Funktionen helfen dabei, allgemeinere Methoden zu erstellen, die die verwendeten und zurückgegebenen Typen genauer darstellen.

function createPair<S, T>(v1: S, v2: T): [S, T] {
  return [v1, v2];
}
console.log(createPair<string, number>('hello', 42)); // ['hello', 42]

TypeScript kann den Typ des generischen Parameters auch aus den Funktionsparametern ableiten.

Klassen

Generika können verwendet werden, um verallgemeinerte Klassen zu erstellen, wie Map.

class NamedValue<T> {
  private _value: T | undefined;

  constructor(private name: string) {}

  public setValue(value: T) {
    this._value = value;
  }

  public getValue(): T | undefined {
    return this._value;
  }

  public toString(): string {
    return `${this.name}: ${this._value}`;
  }
}

let value = new NamedValue<number>('myNumber');
value.setValue(10);
console.log(value.toString()); // myNumber: 10

Aliase

Generische Elemente in Typaliasen ermöglichen die Erstellung von Typen, die besser wiederverwendbar sind.

type Wrapped<T> = { value: T };

const wrappedValue: Wrapped<number> = { value: 10 };

Dies funktioniert auch mit Schnittstellen mit folgender Syntax:interface Wrapped<T>

Standartwert

Generika können Standardwerte zugewiesen werden, die gelten, wenn kein anderer Wert angegeben oder abgeleitet wird.

class NamedValue<T = string> {
  private _value: T | undefined;

  constructor(private name: string) {}

  public setValue(value: T) {
    this._value = value;
  }

  public getValue(): T | undefined {
    return this._value;
  }

  public toString(): string {
    return `${this.name}: ${this._value}`;
  }
}

let value = new NamedValue('myNumber');
value.setValue('myValue');
console.log(value.toString()); // myNumber: myValue

Erweitert

Den Generika können Einschränkungen hinzugefügt werden, um die zulässigen Inhalte einzuschränken. Die Einschränkungen ermöglichen es, sich bei Verwendung des generischen Typs auf einen spezifischeren Typ zu verlassen.

function createLoggedPair<S extends string | number, T extends string | number>(v1: S, v2: T): [S, T] {
  console.log(`creating pair: v1='${v1}', v2='${v2}'`);
  return [v1, v2];
}

Utility Types

TypeScript verfügt über eine große Anzahl von Typen, die bei einigen gängigen Typmanipulationen hilfreich sein können und üblicherweise als Dienstprogrammtypen bezeichnet werden.

Teilweise

Partial Ändert alle Eigenschaften in einem Objekt so, dass sie optional sind.

interface Point {
  x: number;
  y: number;
}

let pointPart: Partial<Point> = {}; // `Partial` allows x and y to be optional
pointPart.x = 10;

Erforderlich

Required Ändert alle erforderlichen Eigenschaften in einem Objekt.

interface Car {
  make: string;
  model: string;
  mileage?: number;
}

let myCar: Required<Car> = {
  make: 'Ford',
  model: 'Focus',
  mileage: 12000 // `Required` forces mileage to be defined
};

Aufzeichnen

Record ist eine Abkürzung zum def eines obj. mit einem Wertyp und schlüsseltyp

const nameAgeMap: Record<string, number> = {
  'Alice': 21,
  'Bob': 25
};

Record<string, number>ist äquivalent zu{ [key: string]: number }

Auslassen

Omit erstellt einen neuen Typ, der alle Eigenschaften eines vorhandenen Typs außer den angegebenen weglässt.

interface Person {
  name: string;
  age: number;
  location?: string;
}

const bob: Omit<Person, 'age' | 'location'> = {
  name: 'Bob'
  // `Omit` has removed age and location from the type and they can't be defined here
};

Wählen

Pick erstellt einen neuen Typ, der nur die angegebenen Eigenschaften eines vorhandenen Typs enthält.

interface Person {
  name: string;
  age: number;
  location?: string;
}

const bob: Pick<Person, 'name'> = {
  name: 'Bob'
  // `Pick` has only kept name, so age and location were removed from the type and they can't be defined here
};

Ausschließen

Exclude erstellt einen neuen Typ, der alle Werte eines vorhandenen Typs ausschließt, die mit dem angegebenen Typ übereinstimmen.

type Primitive = string | number | boolean
const value: Exclude<Primitive, string> = true; // a string cannot be used here since Exclude removed it from the type.

Rückgabetyp

ReturnType erstellt einen neuen Typ, der den Rückgabetyp einer Funktion enthält.

type PointGenerator = () => { x: number; y: number; };
const point: ReturnType<PointGenerator> = {
  x: 10,
  y: 20
};

Parameters

Parameters erstellt einen neuen Typ, der die Parameter einer Funktion enthält.

type PointPrinter = (p: { x: number; y: number; }) => void;
const point: Parameters<PointPrinter>[0] = {
  x: 10,
  y: 20
};

Readonly

Readonly erstellt einen neuen Typ, der alle Eigenschaften eines vorhandenen Typs als schreibgeschützt markiert.

interface Person {
  name: string;
  age: number;
}
const person: Readonly<Person> = {
  name: "Dylan",
  age: 35,
};
person.name = 'Israel'; // prog.ts(11,8): error TS2540: Cannot assign to 'name' because it is a read-only property.

Keyof

keyof ist ein Operator, der alle Schlüssel eines Typs als Union-Typ zurückgibt.

interface Person {
  name: string;
  age: number;
}
// `keyof Person` here creates a union type of "name" and "age", other strings will not be allowed
function printPersonProperty(person: Person, property: keyof Person) {
  console.log(`Printing person property ${property}: "${person[property]}"`);
}
let person = {
  name: "Max",
  age: 27
};
printPersonProperty(person, "name"); // Printing person property name: "Max"

Keyof mit Indexsignaturen

keyof kann auch mit Indexsignaturen verwendet werden, um die Schlüssel eines Objekts zu erhalten.

type StringMap = { [key: string]: unknown };
// `keyof StringMap` resolves to `string` here
function createStringPair(property: keyof StringMap, value: string): StringMap {
  return { [property]: value };
}

deifinitiv typisiert

Für NPM-Pakete im breiten JavaScript-Ökosystem sind nicht immer Typen verfügbar.

Manchmal werden die Projekte nicht mehr gepflegt, und manchmal haben sie kein Interesse daran, sind nicht damit einverstanden oder haben keine Zeit, TypeScript zu verwenden.

Verwendung nicht typisierter NPM-Pakete in TypeScript Die Verwendung untypisierter NPM-Pakete mit TypeScript ist aufgrund fehlender Typen nicht typsicher.

Um TypeScript-Entwicklern bei der Verwendung solcher Pakete zu helfen, gibt es ein von der Community gepflegtes Projekt namens Definitely Typed.

Definitely Typed ist ein Projekt, das ein zentrales Repository von TypeScript-Definitionen für NPM-Pakete bereitstellt, die keine Typen haben.
